# Frontend audit — 2026-04-29

External UX / a11y / privacy / SEO audit of the live `ancap.cloud` frontend, with all
fixes that were applied in the same pass landed against `frontend-app/`.

This document is the **why**. The **what** is in [`LOG.md`](../LOG.md) under the entry
of the same date, and in the diff itself. Read this when you want to know which
problems still need a product or backend decision.

> Status: **15 findings discovered**, **10 landed in this pass**, **5 deferred** (require
> backend, product, or DB-cleanup decisions — see "Deferred" at the bottom).

## How the audit was performed

- 8 surfaces walked while signed in as `3n3rgy`:
  `/`, `/acp`, `/projects`, `/dashboard`, `/wallet/acp`, `/marketplace`, `/agents`,
  `/onboarding`, `/feed`.
- DOM, network requests, and console messages were inspected for each route.
- Findings were cross-referenced with the source under `frontend-app/src/app/`.
- All landed fixes verified with `npx tsc --noEmit`, `npm run lint`,
  `npm run build`, `npm test`.

## Findings landed in this pass

### 1. PII leak on `/agents` — `critical`

`app/agents/page.tsx` rendered `Signed in as ${user.email}` in the page body next to
the agent list. Anyone screenshotting their own agents leaked their email.

**Fix.** Removed the badge entirely; user identity stays in the global header where
it always was.

### 2. Public feed dumped raw payloads — `high`

`/feed` rendered `<pre>{JSON.stringify(payload, null, 2)}</pre>` for every event,
exposing full UUIDs (`run_id`, `strategy_version_id`) on a public route.

**Fix.** Rewrote the page to render readable cards (event label, time, short id,
key payload fields). UUIDs are truncated to `xxxxxxxx…xxxx`. The raw JSON is now
behind an opt-in **"Raw payload"** toggle, off by default.

### 3. Withdraw enabled with `available_acp = 0` — `high`

`/wallet/acp` showed the **Withdraw** button as active even when the available
balance was 0 ACP or the destination address was malformed. Clicking it returned a
generic backend error and gave the user no idea why.

**Fix.** Client-side validation: button disabled until amount is positive, address
matches the bech32 ACP regex, password is filled, and amount ≤ available balance.
Inline error hints under each input. **"In work"** got a hover tooltip explaining
that those funds are reserved by pending orders / escrow / swaps.

### 4. Marketplace showed identical listings as separate cards — `high`

The same `strategy_id` ("Revenue Funnel 20260429-170504") appeared as 6 separate
cards with different prices. No search, no sort, no filter. The **Order note**
textarea sat above all listings and applied globally before any item was selected.

**Fix.** `app/marketplace/page.tsx` rewritten:
- listings are grouped by `strategy_id`, primary card shows the cheapest variant,
  others under a **"Show all N listings"** disclosure;
- added a search box and a sort dropdown (price asc / desc / newest);
- the order note moved into a per-listing modal triggered by **Place Order**, with
  the strategy name and chosen price shown for confirmation;
- per-listing in-flight state, so a slow seller can't grey out the whole page.

### 5. EN/RU/UK was three unrelated buttons — `medium`

In `Navigation.tsx`, the language switcher was three independent `<button>` elements,
so screen readers announced "button" three times instead of "1 of 3 in radiogroup".
Keyboard users had no arrow-key navigation.

**Fix.** Extracted a `LangSwitcher` component with `role="radiogroup"`, `aria-checked`,
roving `tabIndex`, and arrow / Home / End keyboard navigation. One implementation,
used in both desktop and mobile nav.

### 6. Particle background ignored `prefers-reduced-motion` — `medium`

`NetworkBackground.tsx` ran a continuous `requestAnimationFrame` loop on every
authenticated and unauthenticated route, regardless of the user's OS-level
"Reduce motion" setting.

**Fix.** Honor `prefers-reduced-motion: reduce`. With the preference on, draw a
single static frame (so the page doesn't look empty) and skip the rAF loop. Subscribe
to `MediaQueryList` `change` events so toggling it takes effect without reload.

### 7. React-hooks `exhaustive-deps` warnings — `low` (×7)

Seven files had `useEffect`/`useMemo` calling local async loaders without listing
them as deps:

- `app/agents/page.tsx` — `loadAgents`
- `app/access/page.tsx` — `loadGrants`
- `app/contracts/[id]/page.tsx` — `load`
- `app/funds/[id]/page.tsx` — `loadData`
- `app/reputation/page.tsx` — `loadEvents`
- `app/strategies/page.tsx` — `loadData`
- `app/dashboard/seller/page.tsx` — `useMemo` was depending on `balances` but reading
  from `eventsByAgent`.

**Fix.** Each loader is now wrapped in `useCallback` and listed in the effect's
deps. The seller-dashboard `useMemo` now depends on the data it reads. After the
fix `npm run lint` is **0 warnings**.

### 8. No per-route SEO metadata — `medium`

Every route shared `<title>ANCAP Platform</title>` and the same description, with no
`<meta property="og:*">` and no `canonical`.

**Fix.** Added `metadataBase` + a title template (`%s · ANCAP`) + OG/Twitter defaults
in `app/layout.tsx`. Added per-route `layout.tsx` (server components) with their own
`Metadata` for the major surfaces:

| Route          | Title                          | `robots`                |
|----------------|--------------------------------|-------------------------|
| `/marketplace` | Strategy Marketplace · ANCAP   | indexable               |
| `/feed`        | Public Activity Feed · ANCAP   | indexable               |
| `/acp`         | ACP Token & Chain · ANCAP      | indexable               |
| `/projects`    | Architecture · ANCAP           | indexable               |
| `/agents`      | AI Agents · ANCAP              | `noindex,nofollow`      |
| `/strategies`  | Strategies · ANCAP             | `noindex,nofollow`      |
| `/wallet/acp`  | ACP Wallet & Swap · ANCAP      | `noindex,nofollow`      |
| `/dashboard`   | Dashboard · ANCAP              | `noindex,nofollow`      |
| `/onboarding`  | Onboarding · ANCAP             | `noindex,nofollow`      |

The per-route layouts only re-export `children`, so no rendering / hydration changes.

### 9. `agents` scope buttons had no semantic role — `low`

The `My` / `All` toggle was two unstyled buttons.

**Fix.** Added `role="tablist"` + `role="tab"` + `aria-selected`.

### 10. Withdraw / bulk inputs lacked autocomplete hints — `low`

Browsers/password managers couldn't tell which field was the wallet password vs
plain text inputs.

**Fix.** Added `autoComplete="off"` to address fields and `autoComplete="current-password"`
to wallet-password fields. Added `inputMode="decimal"` to the amount input.

## Deferred — needs backend, product, or DB cleanup

These were identified but **not** fixed in this pass because they need decisions
that go past frontend code:

1. **Information architecture / persistent sidebar.** The 30+ flat menu items in
   `Navigation.tsx` should be grouped (Trade · Capital · Identity · Markets · Ops)
   and admin-only items hidden by role. Needs a role-flag from the API and a
   product call on the grouping.
2. **Dashboard KPIs.** `/dashboard` shows hardcoded `1 / 1 / 1` for Agents / Active
   Strategies / Runs. The fix is a real aggregation endpoint, not a frontend tweak.
3. **Wallet password "Unlock for 5 min" session.** Currently every form re-asks for
   the wallet password. To cache a derived key in memory we need an API change
   (the backend currently expects the password per request).
4. **QR for deposit address.** Worth adding once a vetted QR library is vendored —
   not adding a new transitive dep without team review.
5. **Fixture sellers visible on prod marketplace.** Six listings with identical
   `Revenue Funnel 20260429-170504` are seeded test data. The marketplace fix here
   only deduplicates them in UI; the proper fix is a DB cleanup or a `prod_seed`
   filter on the API.
